# DIRECTION — the Creator's editorial taste + routing rubric

This is the **shared, cross-kind** judgement that decides *what to make next* each
run: an **app** (an interactive `/apps` piece) or a **post** (a `/posts` research
note). It is the "editor-in-chief." Execution taste lives elsewhere and is
specialist per-kind:

- apps → `agent/apps/TASTE.md` (visual / interaction / fun / wow)
- posts → `agent/posts/RUBRIC.md` (grounding / insight / clarity, ported from
  stuck-problems-agent's evaluator)

The router applies THIS file; the per-kind judge applies the specialist file.
Lessons from both roll up into the "Lessons" section here over time.

## What belongs on montethakkar.com

A personal site that shows Monte **builds cool things agentically** and **thinks
about hard problems**. Two complementary streams:
- **Apps** — playful, boundary-pushing, fun-to-touch web-dev (the `field`/`shatter`
  lineage). Delight first; personalization optional.
- **Posts** — "important but stalled" research notes on topics worth caring about
  (public health, climate, energy, safety, oceans, space, policy), clearly badged
  AI-generated. Grounded and useful over exhaustive.

## Routing — pick the single most interesting next thing

Each run the router builds a **cross-kind slate** (≈2 app + 2 post pitches),
scores each 1–10 on the dimensions below, and ships the top one. Not a coin-flip,
not a fixed ratio — a judged competition, so "most interesting" is real.

### Routing dimensions (score each pitch 1–10)
1. **Novelty** — fresh vs everything already shipped (`agent/apps/IDEAS.md`,
   `agent/posts/IDEAS.md`, `src/data/apps.ts`, `content/posts/`). Repeats score low.
2. **Interestingness / payoff** — for an app: delight + "how is that done?"; for a
   post: a non-obvious finding worth reading. Would Monte be glad it exists?
3. **Balance** — correct the mix. If the last 2–3 runs were all apps, a strong
   post pitch gets a bump (and vice-versa). Avoid monoculture (5 shaders / 5
   policy briefs in a row).
4. **Timeliness / fit** — does it suit the site right now?
5. **Feasibility in one run** — can a specialist pipeline finish a *good* version
   in a single run (green-gated app, or a grounded ~800–1,500w post)?

Sum, pick the max. Break ties toward the under-represented kind (dimension 3).

## Principles
- **P1. Variety is the point.** Two streams, kept roughly balanced over time.
- **P2. Most-interesting beats on-schedule.** Ship the best pitch, whatever kind.
- **P3. Don't repeat.** Dedup hard against both IDEAS ledgers every run.
- **P4. Quality floor is non-negotiable.** Apps pass the green-gate; posts pass
  the grounding rubric. A weak winner is a no-ship, not a ship.
- **P5. Learn.** Every run appends a one-line lesson below; refine the dimensions
  when a pattern emerges.
