## Soft criteria notes

- Aesthetic: muted sphere + thin outlines + single accent-blue pin color match the site; chrome stays quiet. Good.
- Motion: auto-rotate is smooth and slow (~0.026/400ms), inertial damping on release; not janky. Resume-after-release has a deliberate delay (~1s) which reads as intentional, not a stall.
- Active-country label uses `text-gray-700` (selected) over a `text-gray-400` placeholder; legible, not near-invisible.
- Pin visual emphasis on hover is real but subtle (region diff ~0.019); could be made more obvious, but it is present and the DOM label is the primary affordance.
- A pre-existing floating "N" badge sits bottom-left on all pages (site-wide chrome, not introduced by this feature).

## Overall verdict: PASS

All 9 hard gates pass with demonstrated interaction evidence. Build/static export green; canvas sized and error-free at both widths; Earth is recognizable; auto-rotate, drag-rotate, and pause/resume behave correctly; pins reveal country name + visit date on hover and on mobile tap; all seven countries are server-rendered in a newest-first legend that survives without WebGL; no regressions on / or /about.

## Evaluator notes (non-blocking, for future polish)

- Pause-on-interaction is bound to drag/press only; hover keeps auto-rotating. This matches the GOAL but if a future AC wants hover to pause too, it currently does not.
- Pin hover emphasis is subtle; consider a stronger color/scale delta for the active pin so it reads clearly in a static frame.
- Hover hit detection requires landing near a pin on a moving globe; pins are small targets. Acceptable, but a slightly larger raycast radius would improve discoverability (the dense grid needed multiple passes; a casual user may need a couple tries).
