## Acceptance criteria

### AC1 — Theme re-colors entire site; migration complete. **PASS**
- `data-theme` matches the seeded theme and `body` background matches the
  theme `--bg` on every combo: `data-theme mismatches: 0, bg mismatches: 0`
  (light `rgb(255,255,255)`, dark `rgb(11,13,16)`, sunset `rgb(251,243,234)`).
- Chrome surfaces track the theme: header & mobile tab-bar bg compute to the
  theme bg (dark `oklab(0.158…)`, sunset `oklab(0.968…)`); hamburger panel:
  light `rgb(255,255,255)`/text `rgb(26,26,26)`, dark `rgb(11,13,16)`/text
  `rgb(231,233,236)`, sunset `rgb(251,243,234)`/text `rgb(43,33,26)`; tab-bar
  active uses theme accent (light `rgb(37,99,235)`, dark `rgb(106,168,255)`,
  sunset `rgb(194,65,12)`).
- **Completeness gate:** exact-match sweep of every rendered body element on
  `/`, `/about`, `/posts`, a post page, `/travel` in dark and sunset → "NO exact
  light-literal hits in rendered body" on all 10 page×theme combos. No surviving
  `#ffffff` bg / `#f3f4f6` / `#1a1a1a` fg / `#6b7280` muted / `#e5e7eb` border /
  `#2563eb` blue accent outside the light palette. (A loose ±2 sweep produced 584
  false hits — all dark's `#e7e9ec` fg coincidentally within 2/channel of
  `#e5e7eb`, plus non-painting `<head>` nodes; exact match clears them.)
- Screenshots: `agent/evals/shots/{home,about,travel}-{light,dark,sunset}-{1280,390}.png`,
  `menu-{theme}-390.png`, `tabbar-{theme}-390.png`.

### AC2 — Genuinely no FOUC, on reload and navigation. **PASS**
- With `localStorage.theme` seeded, the first paintable frame is already themed:
  at `waitUntil:"commit"`, `document.documentElement[data-theme]` = the seeded
  value and `body` bg is the themed color — dark `{dataTheme:"dark",
  bodyBg:"rgb(11,13,16)",colorScheme:"dark"}`, sunset `{dataTheme:"sunset",
  bodyBg:"rgb(251,243,234)"}`. The blocking inline head script runs before paint.
- First-frame screenshots show no white flash: `fouc-dark-firstframe.png`
  (fully dark), `fouc-sunset-firstframe.png` (warm cream).
- Navigation `/`→`/travel` with stored non-default theme keeps the theme on the
  first nav frame: `fouc-dark-nav-travel-firstframe.png` is dark with
  "Loading globe…" on dark bg (`{dataTheme:"dark",bodyBg:"rgb(11,13,16)"}`).

### AC3 — OS default before choice; persists across reload + navigation. **PASS**
- No stored theme, emulated `colorScheme:"dark"` → renders dark
  (`data-theme:"dark"`, `bodyBg:rgb(11,13,16)`, `localStorage.theme=null`).
- No stored theme, `colorScheme:"light"` → renders light
  (`data-theme:"light"`, `bodyBg:rgb(255,255,255)`, `localStorage.theme=null`).
- Explicit `sunset` choice with OS=light: survives a full reload
  (`data-theme:"sunset"`), survives client nav `/`→`/travel`→`/`
  (`sunset` throughout) — i.e. the stored choice overrides OS across reload and
  navigation.

### AC4 — Measurable legibility, all themes, both viewports. **PASS**
- `contrast failing rows: 0` (body ≥4.5, muted ≥3, accent ≥3) on home/about/
  travel at both viewports. Computed ratios (color over background):

| theme  | body              | muted             | accent            |
|--------|-------------------|-------------------|-------------------|
| light  | 17.40 (26/26/26)  | 5.43 (95/107/122) | 5.17 (37/99/235)  |
| dark   | 16.00 (231/…/236) | 8.48 (163/172/184)| 8.02 (106/168/255)|
| sunset | 14.31 (43/33/26)  | 5.28 (124/96/71)  | 4.71 (194/65/12)  |

  (Lowest of any metric in any theme is sunset accent 4.71, above the 3:1 bar;
  every body value clears AA 4.5 with large margin.) Travel's "body" sample is
  the muted intro `<p>`, still ≥5.28. Corroborated by screenshots — no
  near-invisible text in any theme.

### AC5 — Globe theme-aware and legible in all three themes. **PASS**
- Distinct recolor per theme — canvas center-region mean RGB: light
  `[140,151,165]`, dark `[4,7,13]`, sunset `[156,142,121]`; cross-theme canvas
  diff light↔dark and dark↔sunset = 100% pixels changed.
- Dark is NOT a pale-blue patch: sphere is dark navy `#1b2735`; borders (light
  blue) and the pin are visible — `globe-dark-1280.png`. Sunset sphere is warm
  tan with visible borders and a distinct orange-red active pin —
  `globe-sunset-1280.png`. Light sphere blue-gray with blue pin —
  `globe-light-1280.png`.
- Active pin is a distinct warm highlight, clearly larger/separate from resting
  pins — `globe-pin-hover.png` (orange enlarged Costa Rica pin among blue pins).
- Label/legend migrated to themed tokens (text-fg/text-muted), legible per AC4.
- **Recolors without reload via the real provider path:** invoking the genuine
  `ThemeProvider.setTheme('dark')` (same API the Sprint-2 UI will call, reached
  via React fiber) live-recolored the globe — sphere center `[143,154,167]` →
  `[2,6,12]`, `data-theme`→"dark", 100% canvas changed, no reload
  (`liveswitch-light.png` / `liveswitch-dark.png`). Note: merely setting the
  `data-theme` attribute *without* React state (not a real code path) recolors
  only the page background and leaves the sphere pale — but that is not how the
  app switches themes; the provider API recolors correctly.

### AC6 — No regressions; clean and green. **PASS**
- Auto-rotation: with no input, canvas changes over time (3s diff
  meanAbsDiff 1.4 / 1.9% changed; 4s diff 1.6 / 2.0%) vs a reduce-motion static
  baseline of exactly 0 / 0% — confirming genuine, ongoing auto-rotation (small

  still auto-rotates by default.
- Drag rotates: before/after a horizontal mouse drag, canvas diff 7.3 / 23.8%.
- Pin reveal: hovering the Costa Rica pin sets the label to

  interaction.
- Sticky header: `header.getBoundingClientRect().top` = 0 before scroll and
  still 0 after `scrollTo(0,800)` (scrollY 742) — sticks.
- No 390px overflow per theme (AC6/gate 4): 0 failures.
- Zero console/page errors per theme on every in-scope page: 0 (incl. WebGL).
- Images naturalWidth>0: no `<img>` elements; nothing broken.
- `pnpm build` exits 0.
