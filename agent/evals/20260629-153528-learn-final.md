## Overall verdict: **FAIL** (one hard-gate-tier AC unmet)

The session loop, IA, variability, personalization, mock/live degradation, themes, and all global gates (build, console, images, overflow, no regressions) PASS with strong evidence. The feature is functionally excellent and shippable in substance. It FAILS one agreed verification item — **#5 Dynamic tab title** — which is a first-class GOAL/SPEC requirement (the "external trigger" that pulls Monte back), not a soft nicety: the streak title is overwritten by static metadata on `/` so the steady-state tab never shows `🔥 N-day streak · Learn`.

## Fixes (ordered by severity)

1. **(Blocking) Make the streak title win on `/`.** `src/app/page.tsx:4` exports `metadata = { title: "Learn" }`, and Next re-applies that static title after hydration, clobbering the effect in `src/components/learn/LearnFeed.tsx:103-110`. Result: `🔥 N-day streak · Learn` flashes then reverts to "Learn" (verified steady-state = "Learn" at 0.5–5s). Fix options: (a) drive the title with Next's API instead of `document.title` (e.g. a `useEffect` that runs after metadata settles / a `<title>` rendered in the client tree), or (b) remove/relax the static `metadata.title` on the root page so the LearnFeed effect is the sole writer, or (c) re-assert the title on an interval / after a microtask so it survives the metadata pass. Expected: on load with a saved streak the tab reads `🔥 N-day streak · Learn` and stays; on complete `New set ready · Learn`; restore a sane base on nav away.
2. **(Minor) Implement pull-to-refresh** in `LearnFeed.tsx` (touchstart/touchmove overscroll at the top of the feed → `load()`), per SPEC §1/§3 and GOAL. Today only the "New set" button refreshes; the named mobile gesture is absent.
3. **(Minor) Add the "offline sample" note** on the degraded path (live → mock fallback in `src/lib/learn-client.ts` catch / no-key). Surface a small note in the feed so the silent degradation is visible, per SPEC §4 / Sprint-4 AC#3.
